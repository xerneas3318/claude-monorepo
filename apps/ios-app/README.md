# ClaudePlanner iOS

Phase 1 read-only checklist. Signs in with Apple, listens to Firestore, renders
`daily/today.md` as a grouped task list. No editing, no voice, no widgets yet
(see `CLAUDE.md` for the phased plan).

## Prerequisites

- macOS with **Xcode 15 or newer** installed from the Mac App Store
  (Command Line Tools alone are not enough — the full Xcode app is required).
- After installing Xcode, run once:
  ```sh
  sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
  xcodebuild -runFirstLaunch
  ```
- `xcodegen` (already installed: `brew install xcodegen`)
- A real iPhone for testing — Sign in with Apple needs a real Apple ID.

## First-time setup

```sh
cd ios-app
./bootstrap.sh           # runs xcodegen, prints any missing pieces
open ClaudePlanner.xcodeproj
```

Then in Xcode:
1. Targets → ClaudePlanner → Signing & Capabilities → pick your Apple Developer team.
2. Build & run on a physical iPhone.

## Firebase setup (do this once)

1. Firebase Console → your project (see `personal.md` for the project ID)
   → Project Settings → Your apps → **Add iOS app** with the bundle id
   from `project.yml`.
2. Download the generated `GoogleService-Info.plist` and save it at:
   ```
   ios-app/ClaudePlanner/GoogleService-Info.plist
   ```
   (this file is gitignored).
3. Firebase Console → Authentication → Sign-in method → **enable Anonymous**.
   (Sign in with Apple would be nicer but it requires a paid Apple Developer
   Program membership. Anonymous gives every install a stable Firebase UID,
   which is enough for a single-user personal app. Swap to a real provider later
   by editing `AuthService.swift`.)
4. Firebase Console → Firestore → Rules. While the sync daemon still writes
   under `userId="default"`, the app reads from `"default"` too. A minimal
   ruleset that allows that for any signed-in user:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{db}/documents {
       match /users/{userId}/{document=**} {
         allow read: if request.auth != null;
         allow write: if false;   // phone is read-only in Phase 1
       }
     }
   }
   ```

## Apple Developer setup

- Free Apple Developer account is enough — sideload to your own iPhone for 7-day
  signing windows.
- No special capabilities needed on the App ID (anonymous auth doesn't require
  Sign in with Apple).

## What this build does

- Firebase init at launch (no analytics — `FirebaseApp.configure()` only).
- Anonymous Firebase Auth (auto sign-in on launch).
- Two Firestore `onSnapshot` listeners: file doc + tasks subcollection,
  ordered by `order`.
- SwiftUI `List` grouped by `category`, checkmark/strikethrough by `checked`.
- Listeners detach when the app enters background and re-attach on foreground
  — matches the no-poll battery rule in `CLAUDE.md`.

## What's intentionally NOT here

Per the phased plan in `CLAUDE.md`:
- Tap-to-toggle (Phase 2)
- Add / edit / move tasks (Phase 3)
- Voice + Claude assistant via Cloud Function (Phase 4)
- FCM push for background updates (Phase 5)
- Widgets, Watch complication (Phase 6)

## Daemon / user-id note

The sync daemon currently writes under `userId="default"` (see
`sync-daemon/src/sync.js`). The iOS app reads from `"default"` too, so the data
lines up out of the box. When the daemon migrates to writing under the
authenticated Firebase UID (Phase 2), change the default in
`TodayStore.init(userId:)` to use `auth.user?.uid` instead.
